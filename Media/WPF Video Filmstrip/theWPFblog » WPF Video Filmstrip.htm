WPF Video Filmstrip

July 05th 2006 Posted to Examples, C# Code, XAML


I've been trying for a while now to write something that would take bitmap snapshots of a video file. After a lot of experimentation I've finally figured it out! At first I tried using the VideoDrawing class and a VisualBrush. This worked great except that it only lets you draw the full-motion video and there is no mechanism for taking a single snapshot. In the end I found the DrawVideo() method of the DrawingContext class which does exactly what I've been after. Here is an an example of that code:

PLAIN TEXT
C#:
  1. RenderTargetBitmap rtb = new RenderTargetBitmap(320, 240, 1/200, 1/200, PixelFormats.Pbgra32);
  2. DrawingVisual dv = new DrawingVisual();
  3. DrawingContext dc = dv.RenderOpen();
  4. dc.DrawVideo(mediaPlayer, new Rect(0, 0, 320, 240));
  5. dc.Close();
  6. rtb.Render(dv);
  7. Image im = new Image();
  8. im.Source = BitmapFrame.Create(rtb);

The example below is a WPF application that takes any WMV video file and draws it out as a series of filmstrips. It updates with a new image every 50 ms. You can enter the URL or path of a video into the address box to load a different video. The performance is pretty damn good as well. Click on the image below to check out the application.
