Xaml Exporter Script for Blender

Daniel Lehenbauer
http://blogs.msdn.com/danlehen

Introduction

This script allows Blender to be used as an authoring tool for 3D Windows Presentation Foundation (WPF) content.  3D scenes created in Blender are exported to .xaml files which can be dynamically loaded or compiled into your WPF applications.  This script currently works with Blender 2.40.  Exported Xaml is compatible with the December CTP of the Windows Presentation Foundation.  For licensing information and to get the latest version go to: http://workspaces.gotdotnet.com/3dtools.

Installation

  1. Ensure you have Blender v2.40 or later installed.  You can view your Blender version via the ~Help~About menu.  The latest version of Blender can be dowloaded from http://www.blender.org.
     
  2. Copy the xaml_export.py in the XamlExporter directory to Blender's scripts directory.  On Windows this is typically located at one of:

    C:\Program Files\Blender Foundation\Blender\.blender\scripts
    C:\Documents and Settings\<YourUserName>\Application Data\Blender Foundation\Blender\.blender\scripts

    depending on the options you chose during installation.  There should be similarly named import and export scripts in this directory.
     
  3. Restart Blender.  If the installation was successful "Xaml (.xaml)..." should now appear in Blender's ~File~Export menu.

(Note: The Xaml Exporter script for Blender works with the Python runtime included with Blender.  A full Python installation is not required.)

Usage

After composing your 3D scene is Blender choose the ~File~Export~Xaml menu.  This will display Blender's standard file dialog.  Enter the name of your scene and press the "Export Xaml" button.

The exported Xaml can be viewed in the WPF Model Viewer (also included in the 3D Tools workspace).  You can dynamically load the Xaml into your WPF application using XamlReader.Load(fileStream).

Known Issues

  • The ImageSource property is exported as a fully qualified path as it appears in Blender (e.g., "D:\MyModels\Ballmer.png").
  • The Windows Presentation Foundation does not have an image codec for Targa (*.tga) files.  If you export a model which uses Targa files you will need to convert these by hand to a format supported by the WPF (e.g., PNG) and fix the exported .xaml by hand.
  • Diffuse materials are exported as a simple texture if applicable or a solid color if no texture is available.  Other types of materials are not currently supported.
  • Blender uses different lighting equations than the WPF.  As a result, exported lighting is approximate.  Only "Sun", "Lamp", and "Spot" light types are supported.
  • There is currently no support for animation or support for geometry types other than meshes.

Please report any issues to the author.  Daniel may be contact through his blog (http://blogs.msdn.com/danlehen).
