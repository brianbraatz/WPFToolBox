Basic Panels - Page 01 - Grid Panel

Grid Panel

Things Worth Knowing About the Grid Panel
  • Unless a specified width or height is set on the grid it will automatically use all the available space.
  • Contents of a Grid panel can be controlled through the use of columns and rows.

    • In my mind the Grid Panel is the Grandaddy of all Panels. It is by far the panel I use the most. Rare do have a layout that doesn’t have a Grid sitting at its root. Here’s a grid in its most simple form holding a button.

    Example 01 - Xaml - Simple Grid With One Control

    <Grid>
       <Button>A Button In A Grid</Button>
    </Grid>

    Example 01 - ScreenShot - Simple Grid With One Control

    The first thing you probably noticed…the button is HUUUGE!!! That’s because a grid panel, by default, uses all the space available to it and in turn, gives it to all of the controls contained within it. In this case it filled my 500 pixel by 500 pixel window and made a 500 pixel by 500 pixel button. The same would occur with a textbox or a rectangle. In fact the Grid Panel is so hardcore about using all the available real estate it will resize itself to fit whatever is holding it. So if I shrink my window down to 200×200 pixels the grid will display a 200×200 button. The same type of behaviour can be seen if I increase my windows size. This can be overridden by giving the Grid’s Width and/or Height property’s a value. Now let’s see what happens when we add another control to our grid.

    Example 02 - Xaml - Simple Grid With Two Controls


    <Grid>
        <Button>A Button In A Grid</Button>
        <TextBox>A TextBox In A Grid</TextBox>
    </Grid>

    Example 02 - ScreenShot - Simple Grid With Two Controls

    You’re probably wondering where the button went. Well, it’s actually behind the textbox. Since the TextBox is the last item in the grid it gets drawn on top of everything else and since we haven’t set up any size constraints it fills the whole grid and consequently covers the button. Of course we could start messing with the width and height properties on the button and textbox and then play around with their margin properties to get them to layout the way we want…but that would be CRAZY!! Not when the Grid Panel let’s us set up columns and rows!!! I mean…it is called a Grid Panel after all.

    Grid Panel - Column and Row Definitions

    Things Worth Knowing About Column and Row Definitions
  • The content of your Grid are assigned to a Column or Row definition as opposed to residing within the definition.
  • A Row’s Height Property is set to * by default. This means it will use as much vertical space as it can get.
  • A Column’s Height Property is set to * by default. This means it will use as much horizontal space as it can get.
    		•

    Column and row definitions are tricky at first. Since it doesn’t matter where you define them in the Grid and you don’t put your content within the tags, they seem to just float ambiguously in free space. Much like I did through out high school. I find it’s best to make all of your row and column definitions at the top of your Grid. I also try and put my controls in the order they are being displayed in the Rows and/or Columns. Without seeing an example your probably wondering what I’m talking about so let me hook you up.

    Example 03 - Xaml - Simple Grid With Two Default Row Definitions


    <Grid>
        <RowDefinition /> <!--Row Number 0 -->
        <RowDefinition /> <!--Row Number 1 -->

        <Button Grid.Row="0" >A Button In A Grid</Button>
        <TextBox Grid.Row="1" >A TextBox In A Grid</TextBox>
    </Grid>

    Example 03 - ScreenShot - Simple Grid With Two Default Row Defintions

    Hopefully things are a little clearer after looking at the example. A couple of things worth noting: The order of row definitions matters. (This also applies to column definitions.) The first is always 0, then next always 1 and so forth. At first this doesn’t seem like a big deal since there doesn’t seem to be anything differing between the two, but it will matter when you start to set the Height property on your rows. The other thing worth noting is that the controls now have their Grid.Row property set. This is how they know which row they should be in. Fairly simple really.

    By default the Height property is *. (This happens to be the same default value Grid has for its Height and Width property as well.) This just means the Row will try and use all leftover vertical space. In our case there are two rows using the * value so they split the available space in half. The same thing occurs for each row added. Here’s an example using the RowDefinition’s Height property.

    Example 04 - Xaml - Simple Grid With Two Row Definitions


    <Grid>
        <RowDefinition Height="Auto" /> <!--Row Number 0 -->
        <RowDefinition Height="*" /> <!--Row Number 1 -->

        <Button Grid.Row="0" >A Button In A Grid</Button>
        <TextBox Grid.Row="1" >A TextBox In A Grid</TextBox>
    </Grid>

    Example 04 - ScreenShot - Simple Grid With Two Row Definitions

    Now our button is a little more under control. By setting the first row’s height property to “auto” I’ve told the row to only be as tall as the content inside wants to be. The visual height you’re seeing is the height the button wants to be.

    Here’s an example where I’ve changed the Rows to Columns. You can see everything works in the same fashion.

    Example 05 - Xaml - Simple Grid With Two Column Definitions


    <Grid>
        <ColumnDefinition Width="Auto" /> <!--Column Number 0 -->
        <ColumnDefinition Width="*" /> <!--Column Number 1 -->

        <Button Grid.Column="0" >A Button In A Grid</Button>
        <TextBox Grid.Column="1" >A TextBox In A Grid</TextBox>
    </Grid>

    Example 05 - ScreenShot - Simple Grid With Two Column Definitions

    And here’s an example where I’ve used both Columns and Rows.

    Example 06 - Xaml - Simple Grid With Both Row and Column Definitions


    <Grid>
        <RowDefinition Height="Auto" /> <!--Row Number 0 -->
        <RowDefinition Height="*" /> <!--Row Number 1 -->

        <ColumnDefinition Width="Auto" /> <!--Column Number 0 -->
        <ColumnDefinition Width="*" /> <!--Column Number 1 -->

        <TextBox Grid.Row="0" Grid.Column="0" >Row 0 Column 0</TextBox>
        <TextBox Grid.Row="0" Grid.Column="1" >Row 0 Column 1</TextBox>
        <TextBox Grid.Row="1" Grid.Column="0" >Row 1 Column 0</TextBox>
        <TextBox Grid.Row="1" Grid.Column="1" >Row 1 Column 1</TextBox>
    </Grid>

    Example 06 - ScreenShot - Simple Grid With Both Row and Column Definitions

    I’ll end this section with a warning. Don’t get caught up trying to make your layouts work in a Grid!!! It’s all too easy to start “tweaking” your columns and rows trying to get things just right. Make sure there isn’t a better panel out there first. I’ve wasted many hours trying to accomplish something in a Grid that would have taken a couple minutes in a DockPanel.
