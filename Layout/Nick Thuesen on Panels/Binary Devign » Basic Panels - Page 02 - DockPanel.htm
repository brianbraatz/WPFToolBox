Basic Panels - Page 02 - DockPanel

Dock Panel

Things Worth Knowing About the Dock Panel
  • The DockPanel docks the controls contained within itself along it’s four sides based on the value of each child’s DockPanel.Dock value.
    		•

    Well…I think the “Things Worth Knowing About the DockPanel” box pretty much summed it up. The DockPanel docks things. Yup…seems like we’re set here. Pretty simple really…ooooor is it? (Insert Knowing Raised Eyebrow Glance Here). Actually, DockPanel is almost that simple. Like that annoying childhood friend, DockPanel will only get along with you if you play by its rules. The rules aren’t too difficult but necessary if you want your content to layout properly. Before we get into those rules let me give you an example of the DockPanel.

    Example 01 - Xaml - Simple DockPanel

    <DockPanel >
       <TextBlock DockPanel.Dock="Top" Background="#9D901B">
          Nasty Brown TextBlock
       </TextBlock>
       <TextBlock DockPanel.Dock="Left" Background="#336699">
          Awesome BlogSite Blue
       </TextBlock>
       <TextBlock Background="White">
          Boring White Filler TextBlock
       </TextBlock>
    </DockPanel>

    Example 01 - ScreenShot - Simple DockPanel

    I know what you’re thinking. “How does he come up with these beautiful designs?” I was given a gift…what can I say?

    Like the Grid panel, the DockPanel will fill all space available to it. In this case the DockPanel has filled my 400×200 pixel window. All but one of the TextBlocks has a DockPanel.Dock property set on them to indicate where the DockPanel should dock them. Obviously DockPanel.Dock=”Top” docks on the top etc etc. The amount of space used by each control depends on where it is docked. Items docked on the Top and Bottom use all available horizontal space while only using the vertical space they need. Conversely, items docked on the Left or Right use all available vertical space while only using the horizontal space they need. If those two sentences were a bit confusing try and apply them to the example. The “Nasty Brown TextBlock” was docked on top so it filled all of the horizontal space it could get and only used enough vertical space to display it’s contained text. On the other hand, the “Awesome BlogSite Blue” TextBlock was docked on the left so it filled all of the vertical space it could get while only using enough horizontal space to display it’s contained text. What about the “Boring White Fill TextBlock”? Well, he’s the filler. He uses all the rest of the space the docked controls leave. Which leads us to a good place to transition into the next section

    DockPanel - Rules of the DockPanel

    Rules Worth Knowing About the DockPanel
    -=ORDER MATTERS=-
  • Unless the DockPanel’s LastChildFill property is set to false then the last child in the DockPanel will automatically fill leftover space even if it has an assigned DockPanel.Dock property.
  • Assigned DockPanel items are stacked on their assigned edge in the order they are listed. The first item on the very edge and the next after it and so on.
  • Assigned DockPanel items listed first get placement priority over other any items listed after them.

    • Above I’ve listed the three main rules I try and keep in the forefront of my mind when I’m working with DockPanel. Since they read like a bad dictionary I’ll discuss each one in the order they appear.
    We already touched a little on the LastChildFill property earlier. In Example 01, the “Boring White Fill TextBlock” fills in the space the first two text blocks leave open. This is because, by default, the DockPanel’s LastChildFill property is True. Look what happens when we explicitly set it to False.

    Example 02 - Xaml - Simple DockPanel With ChildLastFill Set to False

    <DockPanel LastChildFill="false">
       <TextBlock DockPanel.Dock="Top" Background="#9D901B">
          Nasty Brown TextBlock
       </TextBlock>
       <TextBlock DockPanel.Dock="Left" Background="#336699">
          Awesome BlogSite Blue
       </TextBlock>
       <TextBlock Background="White">
          Boring White Filler TextBlock
       </TextBlock>
    </DockPanel>

    Example 02 - ScreenShot - Simple DockPanel With ChildLastFill Set to False

    As you can see the “Boring White TextBlock” doesn’t fill up the extra space anymore. In fact the DockPanel is forcing it to dock even though it doesn’t have DockPanel.Dock property set. The default position for DockPanel.Dock is Left. If we were to remove all of the DockPanel.Dock properties every TextBlock would default to the left side.

    Here’s where a rule come into play, so pay attention. If LastChildFill is not set to false then the last item in your DockPanel WILL FILL THE LEFTOVER SPACE. DockPanel won’t care if you’ve assigned a DockPanel.Dock value or not. This might not seem like such a difficult thing to grasp but if you forget this rule things can become frustrating. Especially for big intense layouts with a lot of things going on. I usually break this rule by attempting to write my xaml intuitively. Example 03 will show what I’m talking about.

    Example 03 - Xaml - Incorrect DockPanel Usage

    <DockPanel>
       <TextBlock DockPanel.Dock="Top" Background="White">
          I'm The Top TextBlock
       </TextBlock>
       <TextBlock Background="White">
          I'm In The Middle
       </TextBlock>
       <TextBlock DockPanel.Dock="Bottom" Background="White">
          I'm The Bottom TextBlock
       </TextBlock>
    </DockPanel>

    Example 03 - ScreenShot - Incorrect DockPanel Usage

    Arrrghh…this is painful for me to even look at. There’s been so many times I’ve made this exact mistake. If you just look at the Xaml everything makes sense. I’ve listed each TextBlock in the order I want it to appear from top to bottom. But obviously they didn’t draw that way. Despite what you might think logically you have to play by DockPanel’s rules. Here’s the proper way to write this Xaml.

    Example 04 - Xaml - Correct DockPanel Usage

    <DockPanel>
       <TextBlock DockPanel.Dock="Top" Background="White">
          I'm The Top TextBlock
       </TextBlock>
       <TextBlock DockPanel.Dock="Bottom" Background="White">
          I'm The Bottom TextBlock
       </TextBlock>
       <TextBlock Background="White">
          I'm In The Middle
       </TextBlock>
    </DockPanel>

    Example 04 - ScreenShot - Correct DockPanel Usage

    This is a great example of why I wrote ORDER MATTERS in the “Rules Worth Knowing” box. Intuitive xaml and DockPanel don’t get along sometimes. Here’s another example of Intuitive Xaml producing an undesired effect.

    Example 05 - Xaml - Incorrect DockPanel Usage

    <DockPanel>
       <TextBlock DockPanel.Dock="Top" Background="White">
          I'm at the top of the "Top" section.
       </TextBlock>
       <TextBlock DockPanel.Dock="Top" Background="White">
          I'm in the middle of the "Top" section.
       </TextBlock>
       <TextBlock DockPanel.Dock="Top" Background="White">
          I'm at the bottom of the "Top" section.
       </TextBlock>

       <TextBlock DockPanel.Dock="Bottom" Background="White">
          I'm at the top of the "Bottom" section.
       </TextBlock>
       <TextBlock DockPanel.Dock="Bottom" Background="White">
          I'm in the middle of the "Bottom" section.
       </TextBlock>
       <TextBlock DockPanel.Dock="Bottom" Background="White">
          I'm at the bottom of the "Bottom" section.
       </TextBlock>

       <TextBlock Background="White">
          I'm the delicious filling in the middle.
       </TextBlock>

    </DockPanel>

    Example 05 - ScreenShot - Incorrect DockPanel Usage

    Unfortunately, like in Example 03, what looks good in XAML doesn’t work on screen. (At least I remembered to put my “filler” at the end this time.) The DockPanel steps through each control contained in markup, from top to bottom, and places each against it’s requested side. That’s why all of our “Bottom” section elements are displayed in reverse order, or I guess I should say, in what “seems” like reverse order. I won’t slam you with another example but if you want to make Example 05 draw with the TextBlock’s in their stated positions, just switch the one that is suppose to be displayed on the top with the one that is suppose to be displayed on the bottom. That way when the DockPanel steps through it’s contents it will reach the TextBlock that declares itself “on the bottom” before the other two and will dock it on the bottom edge before stacking the other two on top of it. Hopefully I won’t have to use the word bottom anymore. I’m starting to blush like a school girl.

    There’s one last “ORDER MATTERS” rule to cover before we move onto to the next panel. It’s not like the others where intuitive Xaml will trip you up, but never-the-less ORDER MATTERS! I’m going to reference Example 01 again so I’ll just re-stick it below so you don’t have to scroll up and down the page.

    Example 01’s Evil Clone- Xaml - Simple DockPanel

    <DockPanel >
       <TextBlock DockPanel.Dock="Top" Background="#9D901B">
          Nasty Brown TextBlock
       </TextBlock>
       <TextBlock DockPanel.Dock="Left" Background="#336699">
          Awesome BlogSite Blue
       </TextBlock>
       <TextBlock Background="White">
          Boring White Filler TextBlock
       </TextBlock>
    </DockPanel>

    Example 01’s Evil Clone - ScreenShot - Simple DockPanel

    Here we have two docked TextBlocks. One docked to the top and one docked to the left. What I want you to notice is the TextBlock docked to the top was given spacial priority first. It doesn’t care what the other TextBlocks are doing and spans the entire horizontal space at top of the panel. The TextBlock on the left only gets to span the vertical space not used by the DockPanel on the top. This is because the TextBlock on the top was listed in our Xaml before the TextBlock on the left. If we switch them we get the following visual.

    Example 01’s Evil Mutated Clone- Xaml - Simple DockPanel

    <DockPanel >
       <TextBlock DockPanel.Dock="Left" Background="#336699">
          Awesome BlogSite Blue
       </TextBlock>
       <TextBlock DockPanel.Dock="Top" Background="#9D901B">
          Nasty Brown TextBlock
       </TextBlock>
       <TextBlock Background="White">
          Boring White Filler TextBlock
       </TextBlock>
    </DockPanel>

    Example 01’s Evil Mutated Clone - ScreenShot - Simple DockPanel

    After using the DockPanel for a while these things become second nature. And now that you know about them you probably won’t have any difficulties. They’re just good things to remember when you’re planning how to lay out your designs.

    Luckily StackPanels are super easy to use.
