Basic Panels - Page 04 - Canvas Panel

Canvas Panel

I’m not gonna lie. I rarely use the Canvas Panel. It’s not that it isn’t useful, it’s just rare that another panel doesn’t fit my needs. I usually find myself using Canvas when all the other panels provide too much layout functionality and I need complete control. That’s what the Canvas Panel offers…oh wait…this should be in a box.

Things Worth Knowing About the Canvas Panel
  • The Canvas Panel offers you complete control in layout using a (x,y) coordinate system to position your controls with.
    		•

    Basically Canvas Panel considers it’s Top-Left corner the coordinate (0,0) and all positions are referenced off that. Here’s an example with buttons.

    Example 01 - Xaml - Simple Canvas With Creepy Button Face

    <Canvas>
       <Button Canvas.Top="0" Canvas.Left="0">
          Left Eye
       </Button>
       <Button Canvas.Top="0" Canvas.Left="150">
          Right Eye
       </Button>
       <Button Canvas.Top="30" Canvas.Left="75">
          Nose
       </Button>
       <Button Canvas.Top="90" Canvas.Left="10">
          Left Smile
       </Button>
       <Button Canvas.Top="120" Canvas.Left="75">
          Middle Smile
       </Button>
       <Button Canvas.Top="90" Canvas.Left="140">
          Right Smile
       </Button>
    </Canvas>

    Example 01 - ScreenShot - Simple Canvas With Creepy Button Face

    This layout would be tricky to replicate in a StackPanel or Grid.

    There is only one thing you have to watch out for when using Canvas. It doesn’t care if its contents are drawn within itself. Let’s set the width and height of the Canvas to 100 pixels and give it that ever-so-cuddly baby blue color as a background so you can visualize what I’m saying.

    Example 02 - Xaml - Simple Canvas With Creepy Button Face Drawing Out of Bounds

    <Canvas Background="LightBlue" Width="100" Height="100" >
       <Button Canvas.Top="0" Canvas.Left="0">
          Left Eye
       </Button>
       <Button Canvas.Top="0" Canvas.Left="150">
          Right Eye
       </Button>
       <Button Canvas.Top="30" Canvas.Left="75">
          Nose
       </Button>
       <Button Canvas.Top="90" Canvas.Left="10">
          Left Smile
       </Button>
       <Button Canvas.Top="120" Canvas.Left="75">
          Middle Smile
       </Button>
       <Button Canvas.Top="90" Canvas.Left="140">
          Right Smile
       </Button>
    </Canvas>

    Example 02 - ScreenShot - Simple Canvas With Creepy Button Face Drawing Out of Bounds

    As you can see the buttons are clearly drawing outside of the Canvas area. Which could be bad if you have other controls drawing there as well. This can be corrected by setting the Canvas’s ClipToBounds property to true like in the following example.

    Example 03 - Xaml - Simple Canvas With Creepy Button Face Drawing Within The Bounds

    <Canvas Background="LightBlue" Width="100" Height="100" ClipToBounds="True" >
       <Button Canvas.Top="0" Canvas.Left="0">
          Left Eye
       </Button>
       <Button Canvas.Top="0" Canvas.Left="150">
          Right Eye
       </Button>
       <Button Canvas.Top="30" Canvas.Left="75">
          Nose
       </Button>
       <Button Canvas.Top="90" Canvas.Left="10">
          Left Smile
       </Button>
       <Button Canvas.Top="120" Canvas.Left="75">
          Middle Smile
       </Button>
       <Button Canvas.Top="90" Canvas.Left="140">
          Right Smile
       </Button>
    </Canvas>

    Example 03 - ScreenShot - Simple Canvas With Creepy Button Face Drawing Within The Bounds

    One last thing about Canvas that is almost a standard in all controls. The order of objects in markup determines the z-order (front of screen to back of screen) controls will be drawn. For example, things at the end of your mark up will be drawn in the front most part of your screen. This is a good thing to remember if you have any objects with overlapping coordinates.

    Now you have a Custom Panel coming your way.
