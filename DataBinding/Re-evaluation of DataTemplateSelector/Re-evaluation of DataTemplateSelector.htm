WPF Programming Challenge #1: Re-evaluation of DataTemplateSelector

I'm writing a WPF app and I want the DataTemplate that gets applied to vary based on the state of the data.

For example, imagine the following ContentControl:

<ContentControl ContentTemplateSelector='...'>
  <ContentControl.Content>
    <l:Person Name='Chris Sells' Age='19' />
  </ContentControl.Content>
</ContentControl>

In my DataTemplateSelector, I can easily look at the entirety of the content to determine which template to apply.

That's great.

Unfortunately, there's no way I can find to tell the ContentControl to “repull” for a new template when the subordinate properties of the content change. AFAICT, the only time my template selector gets repulled is when the Content property itself is set again.

I've already asked the authors of two WPF books for an elegant solution, but came up empty.

Who's going to assert WPF alpha-geek and show us all how it's done?

    I remember this being discussed in the early days and I believe this is now a "known" limitation of the DataTemplate[Selector] API.

    Using DataTemplates directly you can't change the template out (since you *are* the template) and using DataTemplateSelector you can't respond to property changes later. So, basically you're SOL if you need a bound property to change the template.

    I don't have time right this second to experiment, but there's probably some hack you could use to have the DataTemplate that you return from the DataTemplateSelector contain a ContentControl itself which has its Content="{Binding}" and its ContentTemplateSelector set via a DataTrigger which covers the various property changes that you want to watch for.

    Cheers,
    Drew

    The ContentTemplateSelector is really for switching templates based on the *type* of the content, because the template usually references the specific properties of that type.
    I don't see a clean way to do this the way you want. But if you want to vary the appearance of the controls based on the *values* of the properties, I'd built that variation into the template itself using triggers.
    In some cases, this might be tough. For example, suppose you want to show a cigarette box if the Age property is 21 or greater. You might want to add a Boolean property to Person named OkForCigs.

    Another (verbose) option would be to use a MultiBinding on the ContentTemplate property instead of using a DataTemplateSelector.

    <ContentControl.ContentTemplate>
    <MultiBinding Converter="...">
    <Binding RelativeSource="{RelativeSource Self}" />
    <Binding Path="Content.Age" RelativeSource="{RelativeSource Self}" />
    </MultiBinding>
    </ContentControl.ContentTemplate>

    The converter could check the age (the second parameter) and return an appropriate DataTemplate by calling FindResource on the ContentControl (the first parameter).

    Actually, it's quite possible, although ugly.

    It is stated in the comments that the selector will not fire. That is obviously caused by the fact the content is not bound to the desired property, but to the object as a whole.
    Changing this will do the trick.

    <ContentControl Name="cc" ContentTemplateSelector="{StaticResource SelectTheContent}" Content="{Binding Age}">
    <ContentControl.DataContext>
    <l:Person Name='Ruurd Boeke' Age='29' />
    </ContentControl.DataContext>
    </ContentControl>

    As you can see, the content is bound to the property 'age' and thus the contenttemplateSelector will fire when it changes..
    The Datacontext of the contentcontrol is set to the entire object. Now, the trick is to set the datacontext of your datatemplates to that:

    <DataTemplate x:Key="alphaGeek" DataType="{x:Type l:Person}">
    <StackPanel DataContext="{Binding ElementName=cc, Path=DataContext}" Orientation="Vertical" Background="Red">
    <Label>GEEK</Label>
    <TextBlock Text="{Binding Name}"/>
    </StackPanel>
    </DataTemplate>

    This will work. ;-)

    Complete sample:

    <Window.Resources>

    <DataTemplate x:Key="alphaGeek" DataType="{x:Type l:Person}">
    <StackPanel DataContext="{Binding ElementName=cc, Path=DataContext}" Orientation="Vertical" Background="Red">
    <Label>GEEK</Label>
    <TextBlock Text="{Binding Name}"/>
    </StackPanel>
    </DataTemplate>

    <DataTemplate x:Key="nonGeek" DataType="{x:Type l:Person}">
    <StackPanel DataContext="{Binding ElementName=cc, Path=DataContext}" Orientation="Vertical" Background="Yellow">
    <Label>nonGeek</Label>
    <TextBlock Text="{Binding Name}"/>
    </StackPanel>
    </DataTemplate>

    <l:GeekSelector x:Key="SelectTheContent" NonGeekTemplate="{StaticResource nonGeek}" GeekTemplate="{StaticResource alphaGeek}" />

    </Window.Resources>

    <StackPanel>
    <ContentControl Name="cc" ContentTemplateSelector="{StaticResource SelectTheContent}" Content="{Binding Age}">
    <ContentControl.DataContext>
    <l:Person Name='Ruurd Boeke' Age='29' />
    </ContentControl.DataContext>
    </ContentControl>

    <Button Click='changeAge'>Toggle birthday</Button>

    </StackPanel>
    </Window>

    -------------

    it is not code to be proud of, but it does achieve what you wish for.

    I wouldn't use the TemplateSelector, as it requires code behind. Instead, use a 'selector' DataTemplate that hosts the DataTemplates you want to ultimately select from.

    If you need more complex switching logic, add a ViewModel class that wraps the actual data and expose and UI stuff there?

    See code:

    <Window x:Class="DonBoxChallenge.Window1"
    xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
    xmlns:l="clr-namespace:DonBoxChallenge"
    Title="Window1" Height="300" Width="300">
    <Window.Resources>
    <DataTemplate x:Key="dataTemplateSelectorDataTemplate">
    <DataTemplate.Resources>
    <DataTemplate x:Key="ellipseDataTemplate">
    <WrapPanel Width="Auto" Height="Auto">
    <Ellipse Fill="Blue" Width="100" Height="100"/>
    </WrapPanel>
    </DataTemplate>
    <DataTemplate x:Key="squareDataTemplate">
    <WrapPanel Width="Auto" Height="Auto">
    <Rectangle Fill="Blue" Width="100" Height="100"/>
    </WrapPanel>
    </DataTemplate>
    </DataTemplate.Resources>

    <ContentPresenter x:Name="presenter" ContentTemplate="{StaticResource squareDataTemplate}" Content="{TemplateBinding Content}"/>

    <DataTemplate.Triggers>
    <DataTrigger Binding="{Binding Path=Age}" Value="19">
    <Setter TargetName="presenter" Property="ContentTemplate" Value="{StaticResource ellipseDataTemplate}"/>
    </DataTrigger>
    </DataTemplate.Triggers>

    </DataTemplate>
    </Window.Resources>
    <Grid>
    <ContentControl ContentTemplate="{StaticResource dataTemplateSelectorDataTemplate}">
    <ContentControl.Content>
    <l:Person Name='Chris Sells' Age='19' />
    </ContentControl.Content>
    </ContentControl>
    </Grid>
    </Window>
