Using the WPF/C# RichTextBox - Part 2

July 8th, 2007     |     

Following up with my earlier Part I of this series covering the RichTextbox, let's look at more RichTextbox functionality that I used when creating kWordpad.

Before I present some code, as a recap, the name of my RichTextbox control is called MainRichTextBox, and selectionRange refers to the range of text currently selected in our rich textbox:

TextRange selectionRange = new TextRange(MainRichTextBox.Selection.Start, MainRichTextBox.Selection.End);

Detecting Selected Styles
When you position your carat or select some text, the following code helps you to know what exact styles were applied. I explicitly only check for properties that I am interested in, but I am sure similar classes exist for anything else you want also.

Detecting Left/Center/Right Alignment:

if (selectionRange.GetPropertyValue(FlowDocument.TextAlignmentProperty).Equals(TextAlignment.Left))
{
// Do something
}
 
if (selectionRange.GetPropertyValue(FlowDocument.TextAlignmentProperty).Equals(TextAlignment.Center))
{
// Do something
}
 
if (selectionRange.GetPropertyValue(FlowDocument.TextAlignmentProperty).Equals(TextAlignment.Right))
{
// Do something
}

Detecting Bold / Italic / Underline

if (selectionRange.GetPropertyValue(FontWeightProperty).ToString() == "Bold")
{
// Do something
}
else
{
// Do something
}
 
if (selectionRange.GetPropertyValue(FontStyleProperty).ToString() == "Italic")
{
// Do something
}
else
{
// Do something
}
 
if (selectionRange.GetPropertyValue(Inline.TextDecorationsProperty) == TextDecorations.Underline)
{
// Do something
}
else
{
// Do something
}

Detecting Font Family

string selectedFont = selectionRange.GetPropertyValue(FlowDocument.FontFamilyProperty).ToString();

Detecting Font Size

string selectedFontSize = selectionRange.GetPropertyValue(FlowDocument.FontSizeProperty).ToString();

Change Font Color
The approach I used for changing the font color is a little different because I used the WinForms Color Dialog window. The only way I could keep compatibility with the two different Color structs used by WPF (System.Windows.Media.Color) and WinForms (System.Drawing.Color) was by breaking the WinForms Color into its a, r, g, and b values and constructing a new WPF System.Windows.Media Color object and applying that to our selected text's ForeGroundProperty:

Color newColor = Color.FromArgb(a, r, g, b);
SolidColorBrush scb = new SolidColorBrush(Color.FromArgb(a,r,g,b));
 
selectionRange.ApplyPropertyValue(TextElement.ForegroundProperty, scb);

You can see all of the above code in action by looking at my source files for kWordpad. I made only minor changes for this post such as cleaning the code up a bit. If you have any questions, suggestions, or improvements to the snippets I've provided, feel free to post here.

Cheers!
