HyperBar

Home > Samples and Tutorials Gallery - Expression Blend > HyperBar

  • HyperBar.zip

HyperBar

This sample demonstrates a hyperbolic bar of images in a Microsoft® Expression Blend™ application. The bar is a customized Grid control whose columns are resized according to the position of the mouse. A hyperbolic-type function is applied to the coordinates of the mouse pointer to determine the size of each column, making the largest column the one that is under the mouse pointer.

The finished project

This sample highlights the following:

  • Customizing a control
  • Using the DependencyProperty type to define custom control properties
  • Using mouse events
  • Using the Make Visual Brush Resource tool
  • Referencing Extensible Application Markup Language (XAML) elements from code
  • Listening for property changes by using a PropertyChangedCallback delegate

In Action

To see the HyperBar sample in action, open the sample in Expression Blend by using the following steps:

  1. Download the sample project from HyperBar.zip.
  2. Open the project in Expression Blend.
  3. Press F5 to run the sample.

Move the mouse pointer over the row of images to see the images change size in response to the proximity of the mouse. You can use the Zoom and Focus sliders to adjust the size and distribution of the images. The Function Preview element shows you how the HyperBar element will behave when you mouse over it. When the Transparency check box is selected, the transparency of the images is also affected by the proximity of the mouse. The Reflection check box toggles a reflection effect below the HyperBar. The reflection effect was created by using the Make Visual Brush Resource tool in Expression Blend.

UX

In the digital world, user-interfaces (UIs) typically contain elements that are either there at full size or are not there at all—there is no in-between. In the natural world, however, information is continuous and appears at varying levels of interest. The natural world conveys information through color, shape, size, focus, and motion. It is these more natural elements of presentation that represent the spirit of user-experience (UX) for applications that are built on Microsoft Window Presentation Foundation (WPF).

The HyperBar sample is inspired by the experience of vision. The eye produces an image whose central area is sharp and full of information, and which is surrounded by a much larger but less distinct peripheral field. Combined, these areas provide both focus and context. When you direct your eye to an object of interest, you can clearly see the object; you can also sense the object's contextual setting, but without being distracted by it. The eye amplifies the image in proportion to the degree of interest (DOI). Similarly, an ideal application will arrange its UI so that the visual impact of each element maps to the user’s DOI. In the HyperBar sample, the user explicitly tells the application what part of the UI is of interest, by placing the mouse pointer there.

Workflow

In addition to traditional skills, the WPF-generation development team appreciates design, psychology, physiology, mathematics, and physics. These skills map to roles, and not necessarily to individuals. Roles that are relevant to the HyperBar application are: math, programming, and design. The math role consults on how mathematics can be used to enhance UX, and to design the math expressions; the programming role translates the math expressions into code and encapsulates the code inside its own class or function so that it can be easily re-used; the design role styles an application while creatively applying the re-usable math functionality to various UI elements to form the UX.

Software

The functionality of the HyperBar sample is implemented in a custom class called HyperGrid, which derives from the System.Windows.Controls.Grid class. The HyperGrid class is implemented in the HyperGrid.cs file, which is included for you with this sample. You can reuse the HyperGrid class in any Expression Blend project by adding the HyperGrid.cs file to the project, building the project, and then selecting the HyperGrid control from the User Controls tab of the Asset Library (click the Asset Library button in the Toolbox).

The HyperGrid class includes the following properties:

  • Distribution. Represents how many columns to display in the grid.
  • Zoom. Represents the size of the child elements in the individual columns.
  • Center. Represents the position of the mouse pointer.
  • UseTransparency. Determines whether to fade out the child elements on each side of Center.
  • MouseLeaveResetCenter. Determines whether to reset Center to the middle of the grid when the mouse pointer leaves.
  • MouseLeaveResetDistribution. Determines whether to reset to an even Distribution of elements when the mouse pointer leaves.
  • MouseLeaveResetZoom. Determines whether to reset Zoom when the mouse pointer leaves.
  • MouseLeaveResetTransparency. Determines whether to reset UseTransparency to true when the mouse pointer leaves.

These properties use the DependencyProperty type so that you can edit the property values under Miscellaneous in the Properties panel of Expression Blend when you place a HyperGrid element onto the artboard. This means that you can also bind data to the properties of a HyperGrid element. When a property value changes, the columns of HyperGrid are updated in a PropertyChangedCallback delegate method called Update.

In the HyperBar sample, there are two HyperGrid elements on the artboard, named HyperBar and FunctionPreview. The Value properties of the slider controls and the IsChecked property of the Transparency CheckBox control are bound to properties of the FunctionPreview element. Therefore, the appearance of FunctionPreview changes depending on the positions of the slider bars and the state of the checkbox. The appearance of HyperBar does not change unless the mouse cursor is over the element. This behavior is accomplished by using the HyperBar_MouseEnter event in the Window1.xaml.cs file.

The following lines of code represent the Gaussian function that is used in HyperGrid.cs to change the sizes of the columns of the HyperGrid element:

    double exponent = (x * x) / (2 * variance * variance);
    return zoom * Math.Exp(-exponent);

In truth, the function is Gaussian, not hyperbolic. From the point of view of the HyperGrid, the x coordinate is how far across the bar the mouse pointer is. The y coordinate that is returned is in the range [0-1], so the calling logic scales the value to the current height of the UI element. To learn more, open the HyperGrid.cs file to read the code comments.

The custom HyperGrid control works best when the images are similar shapes. They need not be the same size, but their aspect ratios should be as close to 1 as possible (that is, their width and height should be the same). When you run the sample, you will notice that the columns of the FunctionPreview bar do not retain aspect ratios of 1. This is because the columns contain Rectangle elements, which do not stretch uniformly. The HyperBar columns contain Image elements that are set to stretch uniformly in both directions. If you were to set the Stretch property of all the Image elements to Fill, the HyperBar images would stretch like the rectangles in FunctionPreview.
The columns of the custom HyperGrid control can only stretch in height to the maximum height of the control. For this reason, the striped background of the HyperBar is smaller in height than the grid so that the images can appear bigger than the background when the mouse pointer moves over them. Also, both HyperGrid controls in this sample fill an entire row of their containing grid elements so that the HorizontalAlignment and VerticalAlignment properties can be set to Stretch. If those properties were not set to Stretch, the height of the HyperGrid control would shrink to fit the largest child element, and therefore the elements would have no room to grow when the mouse pointer moves over them.
The child elements of a HyperGrid element must be objects that are derived from the FrameworkElement class (for example, Rectangle or Image). This is because the code in the HyperGrid.cs file has to enumerate through the child elements, and the FrameworkElement was chosen and the potential child element type. Additionally, if you have any other child elements in a HyperGrid element, the Panel.ZIndex property of the element must be set to 0 so that it will not be included in the enumeration. For an example, see the BarBackground element in the Window1.xaml.cs file of the HyperBar sample.

Where to go from here

  • To perform an action when the images of the HyperBar element are clicked, you can implement a MouseDown event method for each of the Image elements in the Window1.xaml.cs file.
  • Develop the HyperGrid class further to handle rows as well as columns.
  • Data-bind to the HyperGrid control by following the tutorial Making the Hyperbar Sample Data-bound.
