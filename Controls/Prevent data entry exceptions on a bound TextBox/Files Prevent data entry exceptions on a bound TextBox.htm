Prevent data entry exceptions on a bound TextBox.

Download
File Details
Downloads: 445 File Size: Unknown
Posted By: micwein Views: 756
Date Added: 19 May 2006

When a TextBox is bound, WPF attempts to convert the text value that is entered by users to the same type as the property to which the TextBox is bound. If it can't convert the text value, say when a user enters alpha characters while the bound property type is numeric, WPF raises an exception.

However, you can prevent WPF from raising an exception and, instead, indicate to the user that the entered value is not of the appropriate format by associating the following snippet with the bindings:

<TextBox Name="startPriceTextBox">
  <TextBox.Text>
    <Binding Path="StartPrice" UpdateSourceTrigger="PropertyChanged">
      <Binding.ValidationRules>
        <ExceptionValidationRule />
      </Binding.ValidationRules>
    </Binding>
  </TextBox.Text>
</TextBox>

When entered data can't be converted, this XAML causes a red-line to be placed around the Textbox control.
