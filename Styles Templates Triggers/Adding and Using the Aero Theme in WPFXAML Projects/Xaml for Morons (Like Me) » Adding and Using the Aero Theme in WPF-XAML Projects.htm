Adding and Using the Aero Theme in WPF/XAML Projects

Posted by Matthias Shapiro under WPF Sample , XAML Sample , Aero theme , Applying The Aero Theme
 

I think the new Vista look is super cool.

I’m actually not the kind to blame Microsoft for everything from user frustration to third world poverty, so I like to think that I’m fairly objective about it all. And I think that the Vista look and feel is very spiffy. And I want to use it in my own applications, which means that I need to design my projects in such a way that I’m using the Windows Aero theme as the default style. This is more complicated than I think it should be, so I’ve put together a tutorial that walks through the process step by step.

1. Create a new project in Microsoft Expressions Blend. Close Blend.

2. Open that project in Visual Studio and right click on the “References” section of the project.

Click “Add Reference” and, in the .NET tab of the resulting pop-up, select “PresentationFramework.Aero” and click “OK”.

3. In the References folder, select “PresentationFramework.Aero” and go to the Properties. Change the “Copy Local” property to “true”.

4. Open “App.xaml” and, below the “Resources scoped at the Application level should be defined here.” line, add the following line:

<ResourceDictionary Source=“/presentationframework.aero;component/themes/aero.normalcolor.xaml“ />

(Note: I’m having trouble getting the HTML right so that you could just copy and paste. Just make sure your quotes are actually quotes.)

5. In the menu bar, select Build –> Clean Solution.

6. Select Build –> Build Solution.

7. Run the application (F5). Close the application.

8. Open the project with Blend and the default style for all the objects will be the Aero theme. Start drawing buttons and check boxes and tabbed controls and they should all default to Aero theme style.

Code for this tutorial can be found here: Adding and Using the Aero Theme in WPF/XAML Blend Projects
